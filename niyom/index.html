NIYOM

A structured execution harness for AI output. Every task goes through Plan, Execute, Verify, and Repair — with adversarial verification between generation and acceptance. Not a chatbot. Not a wrapper. Infrastructure. In active development.

The problem it’s built to solve

41–87% of production AI failures are architectural, not model-based. The model isn’t the problem — the harness is. Standard AI tools generate output and present it directly. There is no verification step. Errors arrive confidently. NIYOM separates generation from judgement — the AI generates, a separate adversarial pass judges independently, and the user only receives output that has passed structured verification.

What NIYOM Is

A governed four-stage loop

NIYOM is not a chatbot, not an AI wrapper, not an agent. It is a verification harness: a governed execution environment that forces AI output through four enforced stages before anything reaches the user.

Plan — the AI commits to a structured approach with declared assumptions and a cognitive contract before any execution begins. Execute — the plan runs step by step in enforced format. Verify — a completely separate AI prompt, independent of the execute prompt, audits the output adversarially. It doesn’t know it’s reviewing the same pipeline’s work. Repair — if Verify returns REDO, the output is rewritten to address every identified failure, and Verify runs again.

Only verified outputs enter memory. No unvalidated work accumulates. This is the Stochastic-Deterministic Boundary: generation is stochastic, Verify is the deterministic gate.

Verdicts

What Verify can return

Every task ends with one of four verdicts — not a score, not a rating. A governed decision about whether the output has earned the right to reach the user.

READY
Output passed structured verification. Use it.
REVIEW
Usable, but findings were noted. Check before acting. Acceptance requires friction.
REDO
Critical failure. Output rewrites automatically. Verify runs again.
BLOCKED
Objective cannot be reliably governed as stated. Pipeline does not start.

Current Status

v1.0.0 · Stage 7 in progress

NIYOM is in active development and internal testing. Six of seven development stages are complete. Stage 7 (Semantic Governance) is in progress.

Every completed run is stored to the user's account, protected by row-level security. Smart memory retrieval injects relevant prior runs into new plans, weighted by recency and verdict quality. Clarification routing classifies every ambiguous objective before the pipeline starts — ambiguity is a governance state, not a repair problem.

Niyom is a product. The research programme has run 432 controlled experiments across the architecture.

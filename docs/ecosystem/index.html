Ecosystem Map

One research programme, several working surfaces.

Living Framework is not a single repo or product. It is a body of research and applied systems around reliable AI workflows: context engineering, memory integrity, verification, state management, provenance, observability, and repair.

The simple map

How the pieces relate.

The projects sit at different levels of the same problem: how increasingly capable AI systems can remain reliable over long-horizon work.

LC-OS
The research foundation: papers, failure taxonomy, operating protocols, and the Mahdi Ledger.
AWEN
Field-intelligence layer: watches the AI field, tracks evidence, updates beliefs, and informs strategy.
RITAM
Substrate research: investigates the structural primitives required for reliable cognition systems.
NIYOM
Execution reliability layer: turns planning, verification, repair, and memory into a practical execution harness.
Templates
Practical adoption layer: lightweight artefacts for people using AI tools today.
Books
Public education layer: applied AI guides for practitioners who want structured, useful AI workflows without reading the full research corpus.

Terminology

Current field language first.

The public website should lead with the language the current AI field recognises: reliability architecture, context engineering, memory integrity, state management, verification, observability, provenance, evaluation, agent workflows, and repair. “Governance” remains part of the research vocabulary where precise, but it should not be the only front-door term because it is overloaded by policy, compliance, and regulation.

# Example: Maya Chen — Freelance Strategy Consultant

> Fully filled-in Cowork governance templates for a freelance consultant managing multiple client projects.

---

## About This Example

**Who:** Maya Chen, independent strategy consultant (B2B SaaS, fintech, non-profit)
**Situation:** Managing 3 active client projects simultaneously, each with different rates, budgets, and deadlines
**Main challenge:** Keeping client context separate — wrong client name in a deliverable is a serious professional failure
**Collaboration stage:** Month 5 (established partnership)

---

## The Templates

| File | What It Shows |
|------|--------------|
| [RUNNING-DOCUMENT.md](RUNNING-DOCUMENT.md) | Three active projects with codenames, rates, remaining days, and next actions |
| [PARTNERSHIP-AGREEMENT.md](PARTNERSHIP-AGREEMENT.md) | Consulting-specific operating contract — client boundaries, deliverable authority |
| [TRUTH-PROTOCOL.md](TRUTH-PROTOCOL.md) | Rules for a consulting context: no fabricated benchmarks, no softened recommendations |
| [CANONICAL-NUMBERS.md](CANONICAL-NUMBERS.md) | Day rates, project budgets, deadlines, revenue calculations |
| [FAILURE-RECOVERY.md](FAILURE-RECOVERY.md) | Two documented failures: wrong client name (F4) and unsourced benchmark (F3/F7) |
| [SESSION-START-PROTOCOL.md](SESSION-START-PROTOCOL.md) | Custom session prompts including project switch protocol |
| [FOLDER-STRUCTURE.md](FOLDER-STRUCTURE.md) | Maya's actual folder structure with codename-based organisation |

---

## Key Patterns to Notice

**Codename discipline:** Real client names never appear in deliverable files. MERIDIAN, GREENPATH, VOLTA are used throughout. This is the primary failure prevention for Maya's work — one wrong name in a client document is a serious professional incident.

**Numbers are always sourced:** CANONICAL-NUMBERS.md holds rates and budgets. When Claude needs a benchmark, it asks rather than invents. F-002 happened because this rule wasn't established yet.

**Failures are documented, not hidden:** The Corrections Log in RUNNING-DOCUMENT.md and the full entries in FAILURE-RECOVERY.md show exactly what went wrong, why, and what changed. This is what makes the collaboration improve over time.

**Project switching is explicit:** The `/switch` command was created after F-001. Without it, Claude carried MERIDIAN context into GREENPATH work. With it, every project switch is a deliberate, confirmed action.

## What This Example Demonstrates

A mature (month 5) collaboration with:
- Clear client separation via codenames and project folders
- Rate and budget governance preventing over-servicing
- Two prior failures documented and used to improve the rules
- Custom session start prompts calibrated to Maya's actual workflow
- Folder structure that prevents file divergence across three concurrent projects
